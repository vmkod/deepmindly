# 🧠 DeepMindly

[![GitHub](https://img.shields.io/badge/github-vmkod/deepmindly-blue?logo=github)](https://github.com/vmkod/deepmindly)
-
Локальный AI-трекер продуктивности для Windows. Фоново отслеживает заголовки
активных окон, сохраняет их в SQLite и группирует по темам с помощью
эмбеддингов + K-Means полностью локально.

## Какую проблему решает и для кого

Большинство существующих трекеров требуют, чтобы пользователь вручную размечал категории («Работа», «Отдых», «Соцсети»). На практике за день открывается много разных окон, и категоризировать их вручную быстро надоедает, а готовые автоматические решения часто платные или отправляют личные данные в облако.

**DeepMindly был создан как решение личной проблемы** - когда нужно готовиться к экзаменам, программировать и делать учебные проекты, но время за компьютером незаметно утекает на YouTube, мемы или мессенджеры. Скрипт автоматически находит смысловые группы в истории заголовков и показывает, **сколько времени** реально ушло на каждое занятие.

**Подходит для:**
* **Школьников и студентов**, которым нужно объективно оценить свое время при подготовке к экзаменам или работе над проектами;
* **Всех, кто хочет проанализировать свои цифровые привычки**.


## Как запустить

### Шаг 1: Клонирование проекта
```bash 
git clone https://github.com/vmkod/deepmindly
cd deepmindly
```

### Шаг 2. Установка зависимостей
**Рекомендуется (uv):** Используйте `uv` для быстрой синхронизации и установки всех необходимых библиотек.
```bash
uv sync
```
**Альтернатива (pip/venv)**: Если вы не используете `uv`, следуйте этим шагам:
1. Создайте виртуальное окружение:
```bash
python -m venv .venv
```
2. Активируйте его (для CMD Windows):
```bash
.venv\Scripts\activate
```
3. Установите зависимости:
```bash
pip install -r requirements.txt
```

### Шаг 3. Запуск системы
- С использованием uv:
```bash
uv run main.py
```
- Без uv:
```bash
python main.py
```

### Команды

```bash
# 1. Запустить фоновый сбор данных (Ctrl+C для остановки)
main.py --watch

# 2. Кластеризовать накопленную историю, сохранить снимок за сегодня
main.py --analyze

# 3. Просмотреть снимки по дням и переименовать кластеры
main.py --history

# 4. Открыть визуальный дашборд в браузере
streamlit run dashboard.py
```

В режиме `--history` доступны команды (вводятся и подтверждаются Enter):

| Команда | Действие                          |
|---------|-------------------------------------|
| `n`     | следующий день      |
| `p`     | предыдущий день      |
| `r`     | переименовать кластер  |
| `q`     | выход                                |


## Входные данные и результат

**Вход:**
- заголовки активных окон Windows, полученные через `pywin32` с интервалом
  из `config.yaml`;
- временные метки каждого изменения заголовка (уже пишутся в SQLite
  автоматически при `--watch`).

**Промежуточные данные (SQLite, `data/deepmindly.db`):**
- `window_titles` — очищенный заголовок, время появления, id назначенного кластера;
- `analysis_runs` / `analysis_clusters` — снимок каждого запуска `--analyze`:
  дата, название кластера, число заголовков, суммарное время, типичные заголовки.

**Результат:**
- таблица в консоли (`rich`) и дашборд (`Streamlit`) с тематическими
  кластерами активности, отсортированными по времени: название, число
  заголовков, суммарное время + доля от общего,
  несколько типичных заголовков-примеров на кластер.

## Инструменты и библиотеки

- **Python 3.10+**, менеджер пакетов **uv** (+ `requirements.txt` для pip)
- **pywin32** — опрос активного окна Windows (`win32gui`)
- **sentence-transformers** (`sergeyzh/rubert-tiny-turbo`) — эмбеддинги заголовков
- **scikit-learn** (`KMeans`) — кластеризация
- **numpy** — работа с векторами и расстояниями
- **SQLite** (стандартная библиотека `sqlite3`) — локальное хранилище
- **rich** — форматированный вывод в консоли (таблицы, панели)
- **loguru** — логирование с ротацией файлов
- **Streamlit + pandas** — визуальный дашборд
- **PyYAML** — конфигурация

## Роль ИИ

ИИ используется в двух местах, оба — полностью локальные модели.

1. **Понимание смысла заголовков** — `sentence-transformers` превращает
   каждый заголовок окна в вектор (эмбеддинг), отражающий его смысл. Похожие
   по теме заголовки (например, "GitHub - Pull Request" и "GitLab - Merge
   Request") оказываются близко друг к другу в векторном пространстве.
2. **Группировка** — `K-Means` кластеризует эти векторы в тематические
   группы без заранее заданных категорий.


## Ограничения

- **Только Windows** — сбор данных завязан на `pywin32`/`win32gui`.
- **Названия кластеров без ИИ** — типичный заголовок не всегда
  даёт красивое название, для этого добавлено ручное переименование.
- **Число кластеров фиксировано** (`ai.n_clusters` в конфиге) — K-Means не
  подбирает оптимальное число кластеров автоматически.
- **Один пользователь, один компьютер** — нет синхронизации между
  устройствами, вся история хранится в одном локальном файле SQLite.

## Идеи развития

- **Сравнение дней между собой** — автоматическое сопоставление кластеров по
  дням (например, по близости центроидов) и наглядный тренд роста/падения
  активности во времени
- **ИИ-нейминг кластеров через LLM** — более осмысленные названия вместо
  типичного заголовка
- **Адаптивное число кластеров** — автоподбор `n_clusters` вместо фиксированного значения в конфиге
- **Уведомления** — например, предупреждение при слишком долгом пребывании
  в непродуктивной категории
